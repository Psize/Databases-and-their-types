
### Задание 1. СУБД
## Ответы на вопросы: 
1.1 Лучше всего подойдёт реляционная СУБД (SQL), например PostgreSQL или Microsoft SQL Server. Они гарантируют ACID-транзакции, строгую схему и целостность данных, что критично для финансов.  
1.2 Для лендингов лучше использовать документо-ориентированную NoSQL (например, MongoDB), так как структура данных о лидах может меняться. Для CRM, где важны связи между клиентами и сделками, оптимальна реляционная СУБД (например, PostgreSQL).  
1.3 Подойдёт простая реляционная СУБД, например SQLite, или даже файловое хранилище. Но в рамках компании разумнее использовать уже имеющуюся PostgreSQL, создав отдельную базу или схему.  
1.4 Лучше всего подойдёт графовая СУБД (NoSQL), например Neo4j. Она специально создана для работы со связями и поиска кратчайших путей.  
1.5 Технически можно использовать PostgreSQL для всех задач благодаря поддержке JSONB и расширениям для работы с графами (например, Apache Age). Однако это будет не самым эффективным решением для логистики. Оптимально — гибридная архитектура с несколькими специализированными СУБД.
